# changelog

תיעוד שינויים - כל שינוי שקלוד מבצע יתועד בקובץ CHANGELOG.md עם מספר גרסה.

## Instructions

### הפעלה אוטומטית
**חשוב:** הסקיל הזה צריך להיות פעיל תמיד. בכל פעם שאתה מבצע שינוי בקוד:

1. **לאחר כל שינוי** - עדכן את קובץ `CHANGELOG.md` בתיקיית הפרויקט
2. **מספור גרסאות** - הגדל את מספר הגרסה בפורמט SemVer

### פורמט הקובץ CHANGELOG.md

```markdown
# Changelog - יומן שינויים

## [X.Y.Z] - YYYY-MM-DD

### שינויים
- תיאור השינוי שבוצע
- קבצים שהשתנו: `file1.py`, `file2.py`

### סיבה
למה השינוי נדרש

---
```

### כללי מספור גרסאות (SemVer)
- **Major (X)** - שינוי שובר תאימות, שינוי ארכיטקטורה משמעותי
- **Minor (Y)** - פיצ'ר חדש, תוספת פונקציונליות
- **Patch (Z)** - תיקון באג, שיפור קטן, refactor

### איך לעדכן

1. **קרא את הגרסה הנוכחית:**
   ```
   קרא את תחילת קובץ CHANGELOG.md כדי לראות את הגרסה האחרונה
   ```

2. **החלט על סוג השינוי:**
   - באג fix → הגדל Z
   - פיצ'ר חדש → הגדל Y, אפס Z
   - שינוי גדול → הגדל X, אפס Y ו-Z

3. **הוסף רשומה חדשה:**
   - תאריך נוכחי
   - תיאור ברור בעברית
   - רשימת קבצים שהשתנו

```markdown

כל השינויים בפרויקט DiyurCalc מתועדים כאן.
```
